## 데이터 흐름

### 초기 분석 플로우
1. 사용자: 생년월일시 + 성별 + 달력 종류 입력
2. 클라이언트 → 서버: POST /api/saju/analyze
3. 서버: 입력 검증 → 사주 계산 엔진 호출 → 사주팔자 산출
4. 서버: 시스템 프롬프트(사주 커맨드) + 사주팔자 데이터로 Claude API 호출
5. Claude: 초기 분석 응답 (격국·용신·주별 분석 등)
6. 서버: 세션 스토어에 사주팔자 + 대화 히스토리 저장
7. 클라이언트: 스트리밍으로 응답 표시

### 후속 질문 플로우
1. 사용자: 채팅창에 추가 질문 입력
2. 클라이언트 → 서버: POST /api/chat
3. 서버: 세션에서 사주팔자 + 대화 히스토리 조회
4. 서버: 히스토리에 새 질문 append → Claude API 호출
5. Claude: 응답 (기존 분석 맥락 유지)
6. 서버: 히스토리 업데이트 저장
7. 클라이언트: 스트리밍 표시

## 컴포넌트 책임

### 사주 계산 엔진 (engine/)
- **입력**: 생년월일시·성별·달력 유형
- **출력**: 사주팔자(연·월·일·시주 간지)
- **의존성**: `data/iljin.json`, `data/jeolip.json`, `manse_data.json`(음력 변환 시)
- **특성**: 순수 함수, 외부 API 호출 없음, 결정론적

### Claude API 게이트웨이 (api/claude/)
- **입력**: 사주팔자 데이터 + 대화 히스토리 + 사용자 질문
- **출력**: Claude의 응답 (스트리밍)
- **책임**: 시스템 프롬프트 구성, 토큰 관리, 에러·재시도
- **주의**: 프롬프트에 개인정보(이름 등) 주입 금지

### 세션 관리 (session/)
- 비회원(기본): 쿠키 기반 임시 세션, 24시간 TTL
- 회원(선택): 로그인 세션 + DB 저장 (암호화)

### 프론트엔드 (frontend/)
- 입력 폼: 생년월일시, 성별, 달력 유형
- 채팅 UI: 메시지 목록, 입력창, 스트리밍 표시
- 사주판 시각화 (옵션): 사주팔자를 전통 형식으로 렌더링

## API 엔드포인트 (초안)

| Method | Path | 설명 |
|---|---|---|
| POST | `/api/saju/analyze` | 초기 사주 분석 요청 |
| POST | `/api/chat` | 후속 질문 채팅 |
| POST | `/api/gungham` | 궁합 분석 (2인 이상) |
| GET | `/api/session/:id` | 세션 조회 (사주팔자 + 히스토리) |
| DELETE | `/api/session/:id` | 세션 삭제 |

각 엔드포인트의 요청·응답 스키마는 구현 시 OpenAPI 문서로 정형화.
