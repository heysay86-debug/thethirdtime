## 개요

사주 분석 결과를 RPG 캐릭터 스탯 카드로 변환하는 시스템을 구현한다.
유저가 사주 분석을 마치면 자신의 결과가 RPG 캐릭터 카드로 생성되어 인스타그램/카카오톡으로 공유할 수 있다.
MBTI 결과 공유처럼 바이럴을 노리는 핵심 에셋이다.

## 엔진 데이터 → RPG 스탯 변환 규칙

사주 엔진 출력(`SajuResult`)에서 카드에 표시할 데이터를 추출한다.

### 1. 기본 스탯 (6종, 각 1~20)

| RPG 스탯 | 약어 | 산출 기준 | 엔진 데이터 출처 |
|---|---|---|---|
| **STR** (힘) | 힘 | 신강약 점수 기반. score 0~100 → 1~20 선형 매핑 | `strength.score` |
| **INT** (지력) | 지 | 인성(편인+정인) 개수. 천간4+지지4 중 인성 비율 → 1~20 | `tenGods` 중 편인/정인 카운트 |
| **WIS** (지혜) | 혜 | 관성(편관+정관) 개수 + 화개살 보너스(+3) | `tenGods` 중 편관/정관 + `sinsal` |
| **DEX** (민첩) | 민 | 식상(식신+상관) 개수 + 역마살 보너스(+3) | `tenGods` 중 식신/상관 + `sinsal` |
| **CHA** (매력) | 매 | 재성(편재+정재) 개수 + 도화살/홍염살 보너스(+3) | `tenGods` 중 편재/정재 + `sinsal` |
| **LUK** (운) | 운 | 현재 대운 점수. analysis.score 0~100 → 1~20 | `daeun.periods` 중 현재 대운 |

산출 공식:
- 십성 기반 스탯: `base = (해당 십성 개수 / 8) × 14 + 3` (최소 3, 최대 17)
- 신살 보너스: 해당 신살 존재 시 +3 (최대 20 cap)
- STR, LUK: `score / 100 × 19 + 1` (선형 매핑, 반올림)

### 2. 오행 레지스턴스 (5종, 각 0~100%)

엔진의 `ohengAnalysis.counts` 배열에서 각 오행의 `withJijanggan` 값을 사용.

| 속성 | 표시 | 산출 |
|---|---|---|
| 木 resist | 🌳 | `(木 withJijanggan / 전체 합) × 100` |
| 火 resist | 🔥 | `(火 withJijanggan / 전체 합) × 100` |
| 土 resist | ⛰️ | `(土 withJijanggan / 전체 합) × 100` |
| 金 resist | ⚔️ | `(金 withJijanggan / 전체 합) × 100` |
| 水 resist | 💧 | `(水 withJijanggan / 전체 합) × 100` |

### 3. 클래스명 (격국 → RPG 직업)

| 격국 | 클래스명 | 한줄 설명 |
|---|---|---|
| 정관격 | 질서의 수호자 | 규율과 명예를 따르는 기사 |
| 편관격 | 그림자 기사 | 비정규 전투에 능한 암살자 |
| 정인격 | 현자의 제자 | 지식과 지혜를 탐구하는 학자 |
| 편인격 | 비밀의 마법사 | 비전의 지식을 다루는 술사 |
| 식신격 | 대지의 요리사 | 풍요와 나눔의 달인 |
| 상관격 | 자유의 음유시인 | 재능과 표현의 예술가 |
| 정재격 | 황금의 상인 | 안정적 부를 쌓는 경영가 |
| 편재격 | 모험의 투자가 | 과감한 도전으로 부를 잡는 도박사 |
| 건록격 | 자수성가의 전사 | 스스로의 힘으로 일어서는 투사 |
| 양인격 | 폭풍의 검사 | 압도적 기세의 전투광 |
| 종강격 | 불굴의 방패 | 어떤 역경에도 꺾이지 않는 수호자 |
| 종왕격 | 왕좌의 계승자 | 태어난 순간부터 리더 |
| 종아격 | 천재의 방랑자 | 끝없는 재능의 유랑 예술가 |
| 종재격 | 황금왕 | 재물의 흐름을 타고난 자 |
| 종살격 | 운명의 집행자 | 권력의 정점에 서는 자 |

### 4. 칭호 (신강약 + 격국 상태 조합)

| 조건 | 칭호 접두어 |
|---|---|
| 극강 + 성격 | ★★★ 전설의 |
| 신강 + 성격 | ★★ 숙련된 |
| 중화 | ★★ 균형잡힌 |
| 신약 + 성격 | ★ 성장하는 |
| 극약 | ★ 잠재력의 |
| 파격 (아무 강도) | 🔥 각성한 |
| 약화 (아무 강도) | ⚡ 시련의 |

예시: `★★ 숙련된 비밀의 마법사` (신강 편인격 성격)

### 5. 속성 (일간 오행)

## 구현 범위

이 세션에서는 **카드 생성 코드 (React 컴포넌트 + API)**를 구현한다.

### 파일 구조

```
src/card/
  stat-mapper.ts      ← 엔진 SajuResult → 카드 데이터 변환
  card-data.ts        ← 클래스명, 칭호, 스킬 상수 테이블
  CharacterCard.tsx   ← React 컴포넌트 (react-pdf 또는 canvas)

app/api/saju/card/
  route.ts            ← POST: SajuResult → PNG 이미지 생성 API

tests/
  stat-mapper.test.ts ← 스탯 변환 단위 테스트
```

### 2단계: CharacterCard.tsx

서버사이드 이미지 생성. 두 가지 방법 중 택 1:

**옵션 A: @vercel/og (satori 기반)** — Next.js ImageResponse로 JSX → PNG 변환. 가볍고 빠름.
**옵션 B: canvas (node-canvas)** — 배경 이미지 위에 직접 그리기. 이미지 합성 자유도 높음.

→ 배경 이미지 + 캐릭터 PNG 합성이 필요하므로 **옵션 B (node-canvas)** 권장.

### 3단계: API

```
POST /api/saju/card
Body: { sajuResult: SajuResult }
Response: image/png (1080×1350)
```

### 4단계: 프론트 연동

Zone B 결과 화면 하단에 "내 캐릭터 카드 만들기" 버튼 추가.
클릭 → card API 호출 → 이미지 다운로드 or 공유 시트.

## 절대 하지 말 것

- 캐릭터/배경 이미지를 새로 생성하지 말 것 (기존 에셋만 사용)
