## 핵심 설계: 관계 유형 선택

유저가 본인과 상대의 관계를 직접 선택한다. 관계 유형에 따라 해석 톤과 중점이 달라진다.

### 관계 유형 (2인)

| 코드 | 라벨 | 해석 중점 |
|------|------|-----------|
| `couple` | 연인 / 부부 | 일지(배우자궁) 중심, 정편 재관 교환, 도화살 |
| `parent_child` | 부모 / 자녀 | 인성-식상 관계, 세대 간 오행 흐름 |
| `friend` | 친구 / 동료 | 비겁 관계, 용신 상호 보완, 경쟁/협력 |
| `business` | 사업 파트너 | 재성-관성 교환, 역마, 식상생재 시너지 |
| `boss_sub` | 상사 / 부하 | 관성-인성 구도, 지휘/복종 오행 흐름 |

### 관계 유형 (3인)

- 3인은 쌍별(A-B, B-C, A-C) + 그룹 역학
- 관계 유형: `team` (팀/그룹), `family` (가족), `friends` (친구들)

## 1단계: 궁합 엔진 (`src/engine/gungham.ts`)

### 입력

```typescript
interface GunghamInput {
  persons: {
    name: string;
    birthDate: string;      // YYYY-MM-DD
    birthTime?: string;     // HH:MM (선택)
    gender: 'M' | 'F';
    calendar?: 'solar' | 'lunar';
    birthCity?: string;
  }[];
  relationType: 'couple' | 'parent_child' | 'friend' | 'business' | 'boss_sub' | 'team' | 'family' | 'friends';
}
```

### 출력 구조

```typescript
interface GunghamResult {
  persons: {
    name: string;
    pillars: Pillars;
    dayGanElement: Element;
    strengthLevel: string;
    yongSinPrimary: Element;
  }[];
  
  // 쌍별 분석 (2인이면 1쌍, 3인이면 3쌍)
  pairs: {
    personA: string;
    personB: string;
    
    // 일간 오행 관계
    dayGanRelation: {
      type: '상생' | '비겁' | '상극';
      direction: string;      // "A가 B를 생" 등
      description: string;
    };
    
    // 용신 교환
    yongSinExchange: {
      aToB: 'helpful' | 'neutral' | 'harmful';  // A가 B의 용신에 미치는 영향
      bToA: 'helpful' | 'neutral' | 'harmful';
      description: string;
    };
    
    // 지지 합충형파해
    jijiRelations: {
      type: '합' | '충' | '형' | '파' | '해';
      ji1: string;
      ji2: string;
      position1: string;    // "A일지" 등
      position2: string;
      significance: 'high' | 'medium' | 'low';  // 일지>월지>나머지
    }[];
    
    // 궁합 점수 (0~100)
    score: number;
    rating: '천생연분' | '좋은 인연' | '보통' | '노력 필요' | '주의';
    
    // 관계 유형별 해석
    interpretation: {
      summary: string;
      strengths: string[];
      weaknesses: string[];
      advice: string;
    };
  }[];
  
  // 3인 이상: 그룹 역학
  groupDynamics?: {
    overallScore: number;
    ohengBalance: string;     // 그룹 전체 오행 분포 평가
    keyPerson: string;        // 그룹 중심 인물
    description: string;
  };
}
```

### 핵심 로직

1. **각 인원 개별 분석**: `analyzeSaju()` 호출하여 원국 데이터 확보
2. **일간 오행 관계**: 두 일간의 생극 비교 (기존 ten_gods.ts의 GENERATES/DESTROYS 활용)
3. **용신 교환**: A의 원국 오행이 B의 용신을 강화하는지 판정
4. **지지 합충형파해**: 두 사람의 지지 8자 전체 교차 비교 (기존 relations.ts 활용)
5. **점수 산출**: 용신교환(40점) + 일간관계(20점) + 지지합충(25점) + 신살(15점)
6. **관계 유형별 해석**: relationType에 따라 해석 텍스트 분기

### 참고 코드

- `.claude/commands/saju-gungham.md` — 궁합 분석 프레임워크 (5단계)
- `src/engine/analyze.ts` — analyzeSaju() 호출 패턴

## 2단계: API (`app/api/saju/gungham/route.ts`)

- `POST /api/saju/gungham`
- 입력: GunghamInput (persons 배열 + relationType)
- 출력: GunghamResult JSON
- 엔진만 사용, LLM 호출 없음 (비용 0원)
- rate limit 적용 (기존 패턴)

## 3단계: 프론트엔드

### 진입점

- 메인 메뉴에 "궁합" 메뉴 추가 (또는 기존 GunghamUpsell에서 연결)
- 또는 Zone C 업셀 → 궁합 페이지로 이동

### 입력 UI (`app/gungham/page.tsx`)

#### 0. "좋은 관계란" 프리앰블 (분석 진입 전 필수)

관계 유형 선택 전, 복길이 유저에게 "좋은 관계"의 의미를 생각하게 한다.
대화형(다이얼로그) 또는 나레이션 방식으로 아래 내용을 전달:

> **핵심 메시지 (복길 톤으로 풀어서 전달):**
>
> 1. "좋은 궁합"의 기준은 사람마다 다르다.
>    - 정서적 유대, 안정감을 원하는 사람
>    - 실리적 도움, 성장 자극을 원하는 사람
>    - 편안함을 원하는 사람 vs. 긴장감 속에서 성장하고 싶은 사람
>    → "자네가 생각하는 '좋은 관계'란 무엇인가?"
>
> 2. 지금 본인의 상태에 따라 맞는 상대가 달라진다.
>    - 지금 힘든 시기라면, 안정을 주는 사람이 맞을 수 있다.
>    - 지금 안정적이라면, 자극과 도전을 주는 사람이 오히려 성장을 도울 수 있다.
>    - 같은 상대라도 시기에 따라 관계의 질이 변한다.
>    → "지금 자네의 상태도 함께 고려해야 하네."
>
> 3. 궁합은 고정된 운명이 아니다.
>    - 점수가 높아도 노력 없이 유지되는 관계는 없다.
>    - 점수가 낮아도 서로의 부족함을 이해하면 더 깊은 관계가 될 수 있다.
>    → "숫자에 얽매이지 말고, 이해의 도구로 사용하게."
>
> 4. "이 사람이 당신의 운명이다"라고 단정하는 곳은 조심하게.
>    - 본인의 상태, 배경, 지금 처한 환경을 충분히 이해한 위에서만 그런 판단이 가능하다.
>    - 특히 온라인에서는 대개 반만 믿는 것이 좋다.
>    - 여기서는 그런 단정을 하지 않네. 데이터를 보여줄 뿐, 해석과 판단은 자네 몫이야.

이 프리앰블을 읽고 "준비되었네" 같은 버튼을 눌러야 관계 유형 선택으로 넘어간다.
유저가 이 과정을 건너뛸 수 없다 (redo 모드에서는 축약 가능).

#### 1~5. 입력 흐름

1. **관계 유형 선택** — 카드 형태로 5개(2인) 중 선택
2. **본인 정보** — 이미 사주풀이한 경우 자동 채움 (localStorage)
3. **상대 정보** — 생년월일시, 성별, 이름 입력
4. **3인 모드** — "한 명 더 추가" 버튼
5. **분석 시작** 버튼

### 결과 UI

- 두 사람 사주 나란히 표시 (미니 PillarTable)
- 궁합 점수 + 등급 (원형 게이지)
- 일간 관계 / 용신 교환 / 지지 합충 카드

#### 관점 질문 분기 (결과 데이터 표시 후)

1차 데이터(점수, 오행 관계, 합충 등)를 보여준 뒤, 복길이 2~3개 질문을 한다.
유저의 선택에 따라 **같은 데이터를 다른 관점으로 재해석**한다. LLM 없이 분기 텍스트로 처리.

**관계 유형별 질문 세트:**

모든 질문에 공통 선택지 "잘 모르겠어요"를 마지막에 추가한다.
선택 시 해당 축은 중립(neutral)으로 처리 — 특정 관점 가중 없이 균형 잡힌 해석을 제공.

```
[couple] 연인 / 부부
  Q1: "이 관계에서 무엇을 원하나?"
    - 편안함과 안정
    - 설렘과 열정
    - 함께 성장하는 동반자
    - 잘 모르겠어요
  Q2: "지금 자네의 상태는?"
    - 외롭거나 힘든 시기
    - 안정적인 편
    - 변화를 원하는 시기
    - 잘 모르겠어요
  Q3: "이 사람에게 가장 바라는 것은?"
    - 정서적 위로와 공감
    - 미래를 함께 그릴 수 있는 비전
    - 있는 그대로의 나를 받아주는 것
    - 잘 모르겠어요

[parent_child] 부모 / 자녀
  Q1: "이 관계에서 가장 중요한 것은?"
    - 서로에 대한 이해
    - 신뢰와 존중
    - 솔직한 소통
    - 잘 모르겠어요
  Q2: "지금 이 관계는 어떤 상태인가?"
    - 가깝고 편안하다
    - 거리감이 있다
    - 갈등이 있다
    - 잘 모르겠어요
  Q3: "이 관계가 어떻게 되길 바라나?"
    - 지금처럼 유지
    - 더 가까워지고 싶다
    - 서로 독립적이되 연결된 관계
    - 잘 모르겠어요

[friend] 친구 / 동료
  Q1: "이 사람과의 관계에서 중요한 것은?"
    - 마음이 통하는 교감
    - 서로에게 좋은 영향
    - 함께 있을 때의 즐거움
    - 잘 모르겠어요
  Q2: "이 사람과 얼마나 오래 알았나?"
    - 아직 잘 모르는 사이
    - 어느 정도 알고 있다
    - 오래된 관계
    - 잘 모르겠어요
  Q3: "이 관계에서 걱정되는 것이 있나?"
    - 특별히 없다
    - 가끔 의견 충돌이 있다
    - 거리가 멀어지는 느낌
    - 잘 모르겠어요

[business] 사업 파트너
  Q1: "이 파트너십에서 가장 중요한 것은?"
    - 실력과 전문성 보완
    - 신뢰와 투명성
    - 추진력과 실행력
    - 잘 모르겠어요
  Q2: "지금 사업의 상태는?"
    - 준비/초기 단계
    - 성장 중
    - 어려움을 겪고 있다
    - 잘 모르겠어요
  Q3: "이 사람에게 기대하는 역할은?"
    - 의사결정 파트너
    - 실무 실행자
    - 외부 네트워크/영업
    - 잘 모르겠어요

[boss_sub] 상사 / 부하
  Q1: "이 관계에서 가장 필요한 것은?"
    - 명확한 방향 제시와 피드백
    - 자율성과 신뢰
    - 성과에 대한 공정한 평가
    - 잘 모르겠어요
  Q2: "지금 업무 환경은 어떤가?"
    - 안정적이고 순탄
    - 빠르게 변화 중
    - 압박이 큰 상황
    - 잘 모르겠어요
  Q3: "이 사람과의 관계에서 바꾸고 싶은 것은?"
    - 소통 방식
    - 업무 분담
    - 특별히 없다
    - 잘 모르겠어요
```

**해석 분기 로직:**

같은 데이터(예: 상극 관계)라도 관계 유형 + 유저 선택에 따라 해석이 달라진다:
- couple + 편안함 + 외로운 시기 → "지금은 이 긴장감이 부담될 수 있네. 서로 거리를 두되..."
- couple + 성장 + 안정적 → "이 자극이 오히려 정체된 자네를 깨울 수 있네..."
- business + 추진력 + 초기 단계 → "이 사람의 에너지가 초기 돌파구를 만들 수 있네..."
- boss_sub + 자율성 + 압박 큰 상황 → "상충이 있지만, 위기 상황에서 서로의 빈틈을 메울 수 있는 구조야..."

실제 구현은 축별 해석 문장을 조합하여 생성 (관계유형 5 × 각 Q 3개 = 축별 텍스트 조합).

질문 후 최종 해석:
- 관계별 해석 (강점/약점/조언) — 유저 선택 반영
- "이런 관점에서 보면..." 톤
- 3인: 삼각 다이어그램 + 그룹 역학

### 해석 원칙 (결과 텍스트 생성 시 필수 준수)

1. **점수는 참고치**: "이 점수가 관계의 전부가 아닙니다"를 반드시 명시
2. **맥락적 해석**: 관계 유형(couple/business 등)에 따라 같은 오행 관계도 다르게 해석
   - 상극이 연인에겐 갈등이지만, 사업 파트너에겐 견제와 균형
   - 비겁이 친구에겐 동지지만, 사업에선 경쟁
3. **시기 반영**: 현재 대운을 고려한 해석
   - "지금 시기에는 이런 관계가 특히 도움이 됩니다"
   - "이 시기가 지나면 관계의 성격이 바뀔 수 있습니다"
4. **양면 해석**: 모든 관계에 강점과 약점을 함께 제시, 일방적 좋음/나쁨 판정 금지
