# 보안 및 개인정보 지침

본 문서의 규칙은 **강제 준수 사항**이다. 코드 리뷰·배포 전 체크리스트로 활용.

---

## 1. 시크릿·API 키 관리

### 금지 사항 (절대)
- API 키, DB 비밀번호, JWT 시크릿을 **코드에 하드코딩하지 않는다**.
- `.env`, `credentials.json`, `*.pem`, `*.key` 파일을 **Git에 커밋하지 않는다**.
- Slack, 이메일, 이슈트래커 등에 키를 평문 공유하지 않는다.
- 로그·에러 메시지에 키 값을 출력하지 않는다.

### 의무 사항
- 모든 비밀값은 환경변수(`.env` 로컬 / 배포환경의 secret manager)로 주입.
- 저장소에는 `.env.example`만 포함. 실제 `.env`는 `.gitignore` 대상.
- 배포 환경에서는 AWS Secrets Manager, GCP Secret Manager, Vault 등 사용.
- 키 로테이션 주기 수립 (최소 분기 1회 권장).
- 키 유출 의심 시 **즉시 재발급**, 유출 범위 감사.

### 환경변수 목록 (예시)
```
ANTHROPIC_API_KEY=       # Claude API 호출용
DATABASE_URL=            # DB 접속 문자열
SESSION_SECRET=          # 세션 쿠키 서명용
SENTRY_DSN=              # (옵션) 에러 모니터링
```

### 코드 패턴
```python
# 올바름
import os
api_key = os.environ["ANTHROPIC_API_KEY"]

# 금지
api_key = "sk-ant-api03-..."  # 하드코딩
```

## 2. 개인정보 처리 (한국 개인정보보호법 준수)

### 수집 원칙

**최소 수집**: 서비스 제공에 **필수적인 정보만** 수집.

| 항목 | 수집 필요성 | 비고 |
|---|---|---|
| 생년월일 (양력/음력) | 필수 | 사주 계산 입력 |
| 출생 시각 | 선택 | 시주 계산 시 |
| 성별 | 필수 | 대운 방향 계산 |
| 이름 | 불요 | **수집하지 않음** |
| 이메일 | 회원제 도입 시만 | 계정 식별용 |
| 전화번호 | **수집 금지** | 본인인증 필요 시 외부 서비스 연동 |
| 주소·위치정보 | **수집 금지** | |

### 사주 입력값의 법적 성격
- 생년월일시는 한국 개인정보보호법상 **개인정보에 해당**.
- 다른 식별자(이름·연락처 등)와 결합 시 민감도 증가.
- **익명 사용(비로그인)** 을 기본으로 제공하여 결합 위험 최소화 권장.

### 저장·전송

- **저장 시 암호화**: DB 레벨 암호화(encryption at rest) + 컬럼 암호화(생년월일시).
- **전송 시 암호화**: HTTPS 필수. HTTP는 개발 환경에서도 금지.
- **로그 마스킹**: 로그에 개인정보 노출 금지. 필요 시 해시·마스킹 처리.
  ```
  원본: 1990-02-15 14:30
  로그: ****-**-** **:** (또는 해시값 sha256 앞 8자리)
  ```

### 이용 동의

회원제 또는 저장을 도입하는 경우:
- 개인정보 수집·이용 동의 UI 제공 (선택 동의 항목 명확 구분)
- 개인정보 처리방침 페이지 제작 및 상시 노출
- 수집 항목·이용 목적·보유 기간·제3자 제공 여부 명시
- 만 14세 미만 수집 시 법정대리인 동의 (실무상 14세 미만 가입 제한 권장)

### 보유 및 파기

- 목적 달성 시 지체 없이 파기.
- 회원 탈퇴 시 즉시 파기 (법정 보관 의무 제외).
- 비회원 세션 데이터는 최대 24시간 후 자동 폐기 권장.

### 제3자 제공

- **Anthropic API 호출 시 주의**: 사용자의 생년월일시 등이 프롬프트로 전송됨.
- Anthropic 정책(2025 기준)은 API 입력을 기본적으로 학습에 사용하지 않으나,
  사용자 동의서에 "AI 분석 서비스(Anthropic) 사용" 고지 포함 권장.
- 프롬프트 전송 시 이름 등 추가 식별정보 **포함 금지**.

## 3. 애플리케이션 보안

### 입력 검증
- 생년월일시 포맷·범위 서버측 검증 (1900~2049, 시각 00:00~23:59)
- HTML·JS 삽입 방지 (XSS): 모든 사용자 입력 escape
- Prompt injection 방지: 사용자 입력을 시스템 프롬프트에 직접 concat 금지,
  반드시 구분자 처리 또는 사용자 역할로 분리

### 세션 관리
- 세션 ID는 **암호학적으로 안전한 난수** 사용 (UUID v4 이상 권장)
- 세션 쿠키: `Secure`, `HttpOnly`, `SameSite=Strict` 플래그 필수
- 세션 만료 시간 설정 (유휴 30분, 절대 24시간 권장)

### API 보호
- **Rate limiting**: IP·세션 단위 분당/시간당 호출 제한
  (예: 채팅 호출 분당 10회, 시간당 60회)
- **Claude API 토큰 비용 통제**: 세션당 대화 히스토리 크기 제한, 일정 길이 초과 시 자동 요약·압축
- CORS 정책: 허용 Origin 명시적 지정

### 에러 처리
- 에러 메시지에 스택 트레이스·경로·환경변수 노출 금지
- 사용자에게는 포괄적 메시지, 로그에는 상세 정보 (단 개인정보 마스킹)

### 로깅·모니터링
- 인증 실패, 비정상 입력, 레이트 리밋 초과 등 이벤트 로깅
- 로그 파일 접근 권한 제한
- 보유 기간 정책 (권장: 90일)

## 4. 의존성·인프라

### 패키지
- `requirements.txt` / `package.json` 버전 고정 (lockfile 포함)
- 정기 취약점 스캔: `pip-audit`, `npm audit`, GitHub Dependabot
- 신규 의존성 추가 시 라이선스·보안 검토

### 배포
- HTTPS 강제 (HSTS 헤더)
- 보안 헤더: CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy
- 서버 정보 숨기기 (Server 헤더 제거)
- 방화벽: 관리 포트 외부 노출 금지
- DB 직접 인터넷 노출 금지

## 5. 배포 전 체크리스트

- [ ] `.env`가 `.gitignore`에 포함되어 있는가
- [ ] 커밋 히스토리에 비밀값이 없는가 (`git log -p | grep -i "api_key\|password\|secret"`)
- [ ] 하드코딩된 시크릿이 코드에 없는가
- [ ] 모든 사용자 입력에 검증이 적용되는가
- [ ] 로그에 개인정보가 마스킹되는가
- [ ] HTTPS가 강제되는가
- [ ] Rate limiting이 적용되는가
- [ ] 에러 메시지에 내부 정보가 노출되지 않는가
- [ ] 개인정보 처리방침이 공개되어 있는가 (회원제 도입 시)
- [ ] Anthropic API 호출 시 이름 등 식별정보가 포함되지 않는가

## 6. 사고 대응

**키 유출 시**:
1. 즉시 해당 키 비활성화·재발급
2. 유출 경위 추적 (Git 히스토리, 로그 확인)
3. Git 히스토리에 포함됐다면 `git-filter-repo`로 완전 제거 후 force push
4. 의심 호출 흔적 감사 (비용·로그)

**개인정보 유출 의심 시**:
1. 관련 서비스 일시 중단 또는 영향 차단
2. 유출 범위·원인 조사
3. 한국 개인정보보호위원회 신고 (72시간 이내)
4. 영향 받는 이용자 통지
