# 육효점 결과 화면 UX 개선 지시서

## 1. 현재 문제 진단

### 정보 위계가 뒤집혀 있다

현재 결과 화면은 **구조 정보**(팔궁·세응·지지·육친)를 먼저 보여주고,
**실제 해석**(총론·효사·17개 카테고리)은 클릭 뒤에 숨겨놓았다.

점을 친 사람이 원하는 것은 "내 질문에 대한 답"이다.
팔궁이 어디고 세효가 몇 번째인지는 그 답을 **읽는 전문가용 메타데이터**이지,
답 그 자체가 아니다.

비유: 병원에서 진료 결과를 보여주는데 MRI 촬영 파라미터를 먼저 보여주고,
"소견서 보기" 버튼을 눌러야 진단 결과가 나오는 상황.

### 구체적 문제 목록

| # | 문제 | 위치 |
|---|------|------|
| 1 | 총론(chongron)이 maxHeight: 200으로 잘림 | CompleteView showDetail |
| 2 | 효사(general)가 첫 줄만 표시됨 | `yao.general.split('\n')[0]` |
| 3 | 17개 카테고리 중 4개만 미리보기 | 희망·재물·건강·연애만 표시 |
| 4 | 전체 카테고리를 보려면 효 클릭 필요 | selectedYao 모달 진입 |
| 5 | 카테고리 목록도 maxHeight: 280으로 잘림 | selectedYao 내부 |
| 6 | 결과 첫 화면에 총론이 없음 | showDetail=false 상태 |
| 7 | 3단계 클릭 필요 (결과→상세→효 선택) | 전체 흐름 |
| 8 | 정통 읽기(gua-reading.ts)가 해석 위에 위치 | showDetail 진입 시 |
| 9 | 동효 읽기 규칙 안내가 전문용어 섞여 있음 | changingYao 가이드 텍스트 |

## 2. 정보 위계 재설계

### 원칙: "답 → 근거 → 구조" 순서

사용자가 결과 화면에 도달했을 때, 위에서 아래로 스크롤하면
자연스럽게 얕은 이해 → 깊은 이해로 나아가야 한다.

```
[유저 질문 리마인드]
        ↓
[① 괘 이름 + 총론 전문]          ← 이것이 "큰 그림"
        ↓
[② 핵심 효사 + 동효 읽기 안내]    ← 이것이 "구체적 답"
        ↓
[③ 17개 카테고리 전체]            ← 이것이 "항목별 답"
        ↓
[④ 지괘 총론 (동효 있을 때)]      ← 이것이 "흘러가는 방향"
        ↓
[⑤ 괘 구조도 · 정통 읽기]        ← 전문가용 참고 (접힘 가능)
```

### 동효 개수별 읽기 규칙 (전통 기준)

이 규칙은 gua-data.json의 어떤 데이터를 보여줄지 결정한다.
현재 코드에 이미 안내 텍스트가 있지만, 실제 표시 로직에 반영되지 않았다.

| 동효 수 | 핵심 해석 소스 | 카테고리 소스 |
|---------|---------------|-------------|
| 0개 | 본괘 총론 | 본괘 초효 카테고리 |
| 1개 | **해당 동효의 효사(general) 전문** | 해당 동효 카테고리 |
| 2개 | 두 동효 효사 모두 (위쪽 효사 강조) | 위쪽 동효 카테고리 |
| 3개 | 본괘 총론 + 지괘 총론 | 본괘 총론 기준 |
| 4개 이상 | 지괘 총론 중심 | 지괘 해당 효 카테고리 |
| 6개 전부 | 지괘 총론 | 지괘 초효 카테고리 |

## 3. 화면별 상세 설계

### 3-A. 결과 첫 화면 (현재의 showDetail=false)

**현재**: 카테고리 카드 이미지 + "상세 해석 보기" 버튼
**변경**: 총론과 핵심 효사를 바로 보여준다. 카테고리 카드는 그 아래.

```
┌─────────────────────────────────┐
│  "{유저 질문}"                     │
│                                   │
│  ── 화산여(火山旅) ──              │
│                                   │
│  [총론 전문 — 잘림 없이 전부]       │
│  "火(불)가 山(산) 위에 있다.        │
│   태양이 산 위에 떠 있는 모양       │
│   ...                             │
│   다른 분야의 사람과 교제에          │
│   집중하면 좋다"                   │
│                                   │
│  ── 핵심 풀이 ──                   │
│  (동효 규칙에 따른 효사 전문)       │
│  (동효 0개면: "변화 없이 이 괘의    │
│   뜻 그대로가 답이다" + 총론 재강조) │
│                                   │
│  ── 항목별 풀이 ──                 │
│  [17개 카테고리 전체 — 잘림 없음]   │
│  희망: ...                         │
│  재물: ...                         │
│  궁합: ...                         │
│  결혼: ...                         │
│  ...                              │
│  건강: ...                         │
│  출마: ...                         │
│                                   │
│  ── 흘러가는 괘 (동효 있을 때) ──   │
│  [지괘 이름 + 총론 전문]            │
│                                   │
│  [카드 이미지 저장] [전체 복사]      │
│  [다시 점치기]                      │
│                                   │
│  ▼ 괘 구조 상세 (접힘)             │
│  (팔궁·세응·지지·육친 도표)         │
│  (정통 읽기 텍스트)                 │
└─────────────────────────────────┘
```

### 3-B. "상세 해석 보기" 단계 제거

현재의 2단계(showDetail) 구조를 **1단계로 통합**한다.
결과 화면 = 전체 해석. 별도 진입 없음.

개별 효 클릭(selectedYao) 기능은 유지하되,
메인 화면에서 이미 핵심 효사와 카테고리가 전부 보이므로
"더 알고 싶은 효"를 탭하는 **보조 기능**으로 전환한다.

### 3-C. 동효별 효사 표시 방식

현재: 버튼에 첫 줄만 + 4개 카테고리 미리보기
변경: 효사 general 전문을 펼쳐서 보여준다.

```
── 움직이는 기운 ──

[읽기 안내: 동효 개수에 따른 안내문]

▸ 셋째 기운 → 화뢰서합
  "현실을 직시하고 안정감을 우선하라
   안정감 있는 상황에서 노력하고 행동해야
   결실이 크고 바르게 된다
   ..."
  (전문 표시, 잘림 없음)

▸ 다섯째 기운 → 풍산점
  "스스로 중심 확고하여 희망의 미래가 있다
   ..."
  (전문 표시, 잘림 없음)
```

### 3-D. 카테고리 표시 규칙

**잘림 제거**: maxHeight 제한 삭제.
17개 카테고리 전부 한 화면에 표시.

**괄호 축약 개선**: 현재 `cat.replace(/\(.*\)/, '')`로 괄호 내용을 삭제하는데,
괄호 안 내용이 의미 있는 경우가 많다 (예: "궁합(동료,우정)", "소송(다툼,분쟁)").
→ 두 줄로 표시하거나, 작은 글씨로 괄호 내용도 보여준다.

```
궁합          내가 마음을 바꾸어 좋아지게 된다
동료·우정

시험          불합격, 공부 더 하라
승진·입학·자격
```

또는 한 줄 유지 시:

```
궁합(동료·우정)    내가 마음을 바꾸어 좋아지게 된다
```

### 3-E. 괘 구조 상세 (접힘 섹션)

현재 상단에 있는 팔궁·세응·지지·육친 도표와 정통 읽기 텍스트를
**하단 접힘(accordion) 섹션**으로 이동한다.

기본 상태: 접힘 (닫힘)
탭하면 열림:
- 괘 도표 (世應 · 지지 · 육친)
- 정통 읽기 텍스트 (gua-reading.ts 출력)

이 부분은 "더 깊이 알고 싶은 사람"을 위한 것이므로
메인 해석 흐름을 방해하지 않아야 한다.

## 4. 카드 이미지(다운로드용) 재설계

현재 카드에는 카테고리만 들어간다.
총론이 빠져 있어서 카드만 보면 맥락이 없다.

### 카드 구성 변경

```
┌─────────────────────────┐
│     화산여 (火山旅)       │
│   "{유저 질문}"           │
│                          │
│  [총론 요약 2~3줄]        │  ← 추가
│                          │
│  희망  ...               │
│  재물  ...               │
│  궁합  ...               │
│  ...                     │
│  건강  ...               │
│                          │
│  제3의시간 · 육효점        │
└─────────────────────────┘
```

총론이 길면 전문을 넣기 어려우므로,
카드용으로는 총론의 **핵심 문장 2~3줄**을 발췌한다.
(gua-data.json의 chongron에서 앞 2~3개 \n 단위 문장)

## 5. 텍스트 복사 기능 개선

현재 formatGuaText 함수가 전체 텍스트를 생성하는데,
여기에 정통 읽기(팔궁·세응·지지 정보)도 포함해야 한다.

복사 텍스트 구조:

```
[화산여 · 火山旅]
"{유저 질문}"

── 총론 ──
(chongron 전문)

── 핵심 효사 ──
(동효 규칙에 따른 효사 전문)

── 항목별 풀이 ──
(17개 카테고리 전체)

── 흘러가는 괘 ──
(지괘 이름 + 총론)

── 참고: 괘 구조 ──
(팔궁·세응·육친 정보)
```

## 6. 구현 시 주의사항

### 삭제하지 말 것

- gua-data.json의 총론(chongron) 전문
- gua-data.json의 효사(general) 전문
- 17개 카테고리 전체 (하나도 빠짐없이)
- 카테고리 키의 괄호 안 내용 (동료,우정 / 다툼,분쟁 등)
- 변괘(byeonGua) 이름
- 지괘 총론

### maxHeight 제한 삭제 대상

- 총론 영역: `maxHeight: 200` 제거
- 카테고리 영역: `maxHeight: 280` 제거
- 모바일에서 스크롤이 길어지더라도 내용이 잘리는 것보다 낫다

### 문체 참고

gua-data.json의 원문은 이미 평서문 + 고어체 혼합이다.
이 문체를 유지한다. 코드에서 임의로 요약하거나 의역하지 않는다.
원문 그대로 보여주되, 줄바꿈(\n)은 그대로 유지하여 가독성을 살린다.
