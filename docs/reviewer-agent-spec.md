# MVP 리뷰어 에이전트 사양서

최종 업데이트: 2026-04-16

---

## 목적

사주웹 MVP의 API 응답을 자동으로 평가하여 **"결제 모듈만 붙이면 상업화 가능"** 수준인지 판정한다.
테스트 사주 3~4건을 API에 전송하고, 응답을 3개 페르소나로 리뷰한 뒤 점수표 + 코멘트를 출력한다.

---

## 구조

```
[테스트 사주 데이터]
       │
       ▼
[Node.js 테스트 하네스]
   ├─ POST /api/saju/analyze  → Phase 1 응답 수집
   └─ POST /api/saju/interpret → Phase 2 SSE 스트림 수집
       │
       ▼
[응답 JSON 병합]
       │
       ├─ [A] 계산 정확성 검증 (프로그래밍 로직, LLM 불필요)
       │
       └─ [B] Claude API 리뷰 호출 (3 페르소나)
              ├─ 일반인 페르소나 (가중치 50%)
              ├─ 중급자 페르소나 (가중치 20%)
              └─ 경쟁사용자 페르소나 (가중치 30%)
       │
       ▼
[종합 리포트 출력] → reviewer-report-{timestamp}.md
```

---

## A. 계산 정확성 검증 (자동화, LLM 불필요)

사주 엔진의 4기둥 산출 정확성은 LLM 판단이 아니라 레퍼런스 대조로 검증한다.

### 검증 항목

| # | 항목 | 판정 기준 | 합격선 |
|---|------|----------|--------|
| A1 | 4기둥(연·월·일·시주) 간지 | 사용자 제공 레퍼런스와 완전 일치 | **100% 필수** |
| A2 | 입춘 경계 처리 | 입춘 전후 출생 시 연주 전환 정확 | 100% |
| A3 | 월주 절입 경계 | 12절 기준 월주 전환 정확 | 100% |
| A4 | 야자시 처리 | 23:00~23:59 → 당일 일주, 시주는 익일 子시 기준 | 100% |
| A5 | 음양력 변환 | 음력 입력 시 양력 변환 후 올바른 간지 산출 | 100% |
| A6 | 대운 산출 | 대운 시작 나이·간지 순서 정확 | 100% |
| A7 | 십성(十星) 배치 | 일간 기준 8위치 십성 정확 | 100% |

### 검증 방식

```typescript
// 의사코드
for (const testCase of TEST_CASES) {
  const response = await fetch('/api/saju/analyze', { body: testCase.input });
  const { engine } = await response.json();

  assert(engine.yearPillar.gan === testCase.expected.yearPillar.gan);
  assert(engine.yearPillar.ji  === testCase.expected.yearPillar.ji);
  // ... 월주, 일주, 시주, 십성 모두 대조
}
```

**A 파트 하나라도 실패하면 전체 리뷰 FAIL.** 계산이 틀린 상태에서 해석 품질을 평가하는 것은 무의미하다.

## B. 페르소나별 LLM 리뷰

Phase 1(핵심 판단) + Phase 2(전체 해석) 응답 전문을 Claude API에 넘겨 페르소나별로 리뷰한다.

### 공통 리뷰 컨텍스트 (시스템 프롬프트에 포함)

```
당신은 한국의 사주 웹서비스 MVP를 리뷰하는 전문 평가자입니다.
아래 JSON은 사주분석 API의 실제 응답입니다.

[서비스 배경]
- 서비스명: 제3의시간
- 이론 기준: 이석영 사주첩경 1차, 5유파 보강 2차
- 타겟: 사주에 관심 있는 일반 한국인 (20~40대)
- 차별점: 격국·용신까지 제시하는 전문 분석을 대중적으로 전달

응답 JSON을 분석하여 아래 평가 기준에 따라 점수(0~100)와 구체적 코멘트를 작성하세요.
점수 근거를 반드시 명시하세요. 모호한 칭찬이나 추상적 비판은 금지합니다.
```

### 페르소나 1: 일반인 (가중치 50%)

**역할 프롬프트:**
```
당신은 사주를 처음 접하는 30대 직장인입니다.
"격국", "용신", "신강" 같은 용어를 모릅니다.
친구가 추천해서 처음 써봤고, 결과를 보고 재미있으면 유료 결제를 고려합니다.
```

**평가 항목:**

| # | 항목 | 평가 질문 | 배점 |
|---|------|----------|------|
| B1-1 | 첫인상 이해도 | summary를 읽고 내 사주가 어떤 성향인지 3초 안에 파악 가능한가? | 25 |
| B1-2 | 용어 접근성 | 전문용어가 설명 없이 등장하는 곳이 있는가? 각 용어에 쉬운 부연이 있는가? | 25 |
| B1-3 | 재미·몰입 | 결과를 끝까지 읽고 싶은가? 중간에 이탈하고 싶은 지점이 있는가? | 20 |
| B1-4 | 행동 유도 | "그래서 나는 어떻게 하면 좋지?"에 대한 실용적 조언이 있는가? | 15 |
| B1-5 | 유료 전환 의향 | 이 무료 결과를 보고 더 자세한 분석에 돈을 낼 의향이 생기는가? | 15 |

**출력 포맷:**
```json
{
  "persona": "일반인",
  "scores": { "B1-1": 75, "B1-2": 60, ... },
  "total": 68,
  "이탈_포인트": ["strengthReading 첫 문장에서 '편인격·신강'이 설명 없이 등장"],
  "개선_제안": ["summary에 한줄 비유 추가: '물 위에 떠다니는 배 같은 사주입니다'"],
  "유료전환_판단": "현재 상태로는 전환 어려움. 무료 결과가 이미 전문적이라 유료의 추가 가치가 불명확"
}
```

### 페르소나 2: 명리학 중급자 (가중치 20%)

**역할 프롬프트:**
```
당신은 명리학 서적을 3~4권 읽은 학습자입니다.
격국, 용신, 신강/신약의 개념을 알고, 자신의 사주를 직접 분석해본 적 있습니다.
기존에 사용하던 무료 만세력 사이트의 결과가 너무 단순해서 대안을 찾고 있습니다.
이석영 사주첩경, 자평진전, 적천수 정도는 이름과 기본 관점을 알고 있습니다.
```

**평가 항목:**

| # | 항목 | 평가 질문 | 배점 |
|---|------|----------|------|
| B2-1 | 격국 판정 납득성 | 제시된 격국이 4기둥 구성으로 보아 납득 가능한가? 근거가 명시되어 있는가? | 30 |
| B2-2 | 용신 논리 일관성 | 신강/신약 판정과 용신이 논리적으로 일관되는가? 모순은 없는가? | 25 |
| B2-3 | 해석 깊이 | 기존 무료 사이트 대비 의미 있는 깊이가 있는가? | 20 |
| B2-4 | 이론 출처 명시 | 유파별 관점이 "적천수 관점에서는~" 형태로 명확히 구분되는가? | 15 |
| B2-5 | 누락 항목 | 형충파해합, 공망, 지장간, 십이운성 등 기대 항목이 빠져 있는가? | 10 |

**출력 포맷:**
```json
{
  "persona": "중급자",
  "scores": { "B2-1": 85, "B2-2": 80, ... },
  "total": 82,
  "이론_오류": ["월지 酉와 일지 戌의 관계를 '해'로 기술했으나 실제로는 해당 없음"],
  "누락_항목": ["공망 언급 없음", "지장간 분석이 주별 분석에 포함되지 않음"],
  "차별점_인정": ["격국 판정 근거를 투출법 기준으로 명시한 점은 타 서비스에 없는 강점"]
}
```

### 페르소나 3: 경쟁 서비스 사용자 (가중치 30%)

**역할 프롬프트:**
```
당신은 청월당, 신한생명 사주, 운세의신 등 기존 사주 서비스를 모두 써본 파워유저입니다.
각 서비스의 장단점을 잘 알고, 새로운 서비스가 나오면 비교 리뷰를 합니다.
UX 감각이 있고, "이 서비스가 돈을 벌 수 있는가"를 판단할 수 있습니다.

[경쟁 서비스 특징 - 평가 기준점]
- 청월당: 캐릭터 가이드 → 조견표(무료) → 대운 → 오행분포 → 희신기신 → 유료 상세
  강점: 진입장벽 낮음, 정형화된 포맷, 빠른 결과 표시
- 일반 만세력 사이트: 4기둥 + 십성표 제공. 해석 없음. 무료.
- 운세 앱(토정비결류): 재미 위주, 학술적 깊이 없음, 광고 수익 모델
```

**평가 항목:**

| # | 항목 | 평가 질문 | 배점 |
|---|------|----------|------|
| B3-1 | 무료/유료 경계 설계 | 무료로 주는 양과 유료 전환 유인의 균형이 적절한가? | 25 |
| B3-2 | 결과 도달 속도 | 입력부터 핵심 결과까지의 체감 속도가 경쟁사 대비 어떤가? | 20 |
| B3-3 | 차별화 인식 | 기존 서비스에 없는 명확한 차별점이 있는가? 사용자가 인식할 수 있는가? | 25 |
| B3-4 | 완성도 | "아직 미완성 같다"는 느낌을 주는 구멍이 있는가? | 20 |
| B3-5 | 재방문 의향 | 이 서비스를 즐겨찾기에 추가하고 다른 사람에게 추천할 의향이 있는가? | 10 |

**출력 포맷:**
```json
{
  "persona": "경쟁사용자",
  "scores": { "B3-1": 55, "B3-2": 70, ... },
  "total": 63,
  "구멍_목록": ["로딩 중 아무 피드백 없음", "결과 페이지에 공유 버튼 없음"],
  "vs_청월당": { "우위": ["격국·용신 깊이"], "열위": ["진입 속도", "캐릭터 몰입감"] },
  "상업화_판정": "현재 상태로는 '베타' 수준. 무료 결과의 범위 조정 + 로딩 UX 개선 후 출시 가능",
  "킬러피처_제안": ["사주 한줄 요약 카드(SNS 공유용)", "비슷한 사주의 유명인 매칭"]
}
```

## 종합 점수 산출

```
종합점수 = (일반인 total × 0.5) + (중급자 total × 0.2) + (경쟁사용자 total × 0.3)
```

### 합격 기준

| 등급 | 점수 | 판정 |
|------|------|------|
| 🟢 출시 가능 | 80점 이상 + A파트 전원 PASS | 결제 모듈 붙이고 출시 |
| 🟡 조건부 출시 | 65~79점 + A파트 전원 PASS | 리포트의 개선 제안 항목 처리 후 출시 |
| 🔴 재작업 | 64점 이하 또는 A파트 FAIL | 근본적 수정 필요 |

## 테스트 데이터 포맷

사용자가 제공할 테스트 케이스 형식:

```json
{
  "id": "test-01",
  "label": "설명 (예: 1990년 2월 15일 오후 2시 30분 남성)",
  "input": {
    "birthDate": "1990-02-15",
    "birthTime": "14:30",
    "calendar": "solar",
    "gender": "M"
  },
  "expected": {
    "yearPillar": { "gan": "庚", "ji": "午" },
    "monthPillar": { "gan": "戊", "ji": "寅" },
    "dayPillar": { "gan": "壬", "ji": "辰" },
    "hourPillar": { "gan": "丁", "ji": "未" },
    "tenGods": {
      "yearGan": "편인", "yearJi": "...",
      "monthGan": "...", "monthJi": "...",
      "dayJi": "...",
      "hourGan": "...", "hourJi": "..."
    }
  }
}
```

> 참고: `expected`의 4기둥 간지는 검증된 만세력에서 추출한 정답이어야 한다.
> 십성은 선택 사항이나, 포함하면 A7 검증도 수행한다.

## 실행 방식

```bash
# 1. 서버 기동 상태에서
npm run dev

# 2. 리뷰 스크립트 실행
npx ts-node scripts/reviewer-agent.ts --cases test-cases.json

# 출력: docs/reviewer-report-20260416-143000.md
```

### 스크립트 동작 순서

1. `test-cases.json` 로드
2. 각 케이스별 `/api/saju/analyze` POST → Phase 1 응답 수집
3. 각 케이스별 `/api/saju/interpret` POST → Phase 2 SSE 스트림 수집 → 섹션 JSON 파싱
4. **A 파트**: 4기둥·십성 레퍼런스 대조 → PASS/FAIL
5. A 파트 전원 PASS인 경우에만 B 파트 진행
6. **B 파트**: Phase1 + Phase2 응답을 Claude API에 전송 (페르소나별 3회 호출)
7. 점수 집계 + 종합 리포트 생성
8. `docs/reviewer-report-{timestamp}.md` 저장

### 비용 추정

- B 파트 Claude 호출: 테스트 케이스 4건 × 페르소나 3개 = 12회 호출
- 모델: claude-haiku-4-5 (비용 효율)
- 예상 입력: ~3,000 tokens/회, 출력: ~800 tokens/회
- 총 예상: 입력 ~36K + 출력 ~10K tokens ≈ $0.04 이하
